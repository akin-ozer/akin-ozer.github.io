SRE Challenge 1

Published at 10.01.202020 min read

Welcome to my series about SRE/DevOps challenges. These posts are real-world based examples about job challenges that happens in recruitment process. Posts aim solving job challenges by examples, these are mainly best practise based solutions and easy to run.

People loves code that just runs, even that involves running script. Try to aim have less steps to run code but don't hardcode secrets in repo. Don't use files, environment variables, binaries in your computer. People should download source code and should be able to run script, that script should care about dependencies and secrets.(maybe ask user for secrets and configuration in script as input or just create secret password file for vaults after download.)

  • Task 1


    For a given password file with the content as following:

    Make a Python program that prints out the duplicate users with their IDs:

    $ ./dupids.py [passwd_file]
100: user1, user3

    Can you think of linux one-liner to display only the duplicated IDs?

    $ <some linux/unix commands>
100

  • Task 2


    Create an account on Google Cloud (https://cloud.google.com). You will have access to all GCP products with $300 credit for free, which is valid for 12 months. There will be no autocharge after free trial ends, you will not be charged unless you manually upgrade to a paid account.

    Install Google Cloud SDK: https://cloud.google.com/sdk/docs/downloads-interactive

    Create a production-ready PostgreSQL server and

    • Set postgres user password to a8dX46JRHwgNVMq8

    • Make sure that it can be reached externally from anywhere

    Create a zonal Kubernetes cluster with 2 nodes and simplest CPU/memory configuration. You can use google web console interface.

    Create a very simple Python application which is supposed to have 2 endpoints: (You may use Flask for simplicity https://flask.palletsprojects.com/en/1.1.x/quickstart)

    • /           : Returns “HTTP 200” with string “Welcome to my app”

    • /status : Returns “HTTP 204” without any content.

    You need to dockerize this application and deploy it to the kubernetes cluster you created earlier.

    Enable Google Container Registry to push your Docker image.

    You may deploy the application manually using gcloud commands or any other deployment tools (e.g. circleci, ansible, gitlab pipeline, bitbucket pipeline, …)

    An external address needs to be assigned to the application running on Kubernetes. Make sure below URLs work:

    • https://$IP_ADRESS/

    • https://$IP_ADRESS/status
